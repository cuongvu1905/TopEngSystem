# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement system-wide Dark/Light Mode with a toggle switch below the "Change Password" button in the profile dropdown menu, preserving current Light Mode exactly and preventing screen flashes on reload.

**Architecture:** Utilize a root HTML `data-theme="dark"` attribute selector to override colors defined as CSS Custom Properties in globals.css. Pre-inject an inline script in layout.js `<head>` to query localStorage and apply `data-theme` immediately before React component mount.

**Tech Stack:** Next.js, React, CSS Custom Properties (Variables), localStorage.

## Global Constraints
- Light Mode must remain identical to the current design (no overrides when `data-theme` is not set).
- Dark Mode toggle states must persist in local storage.
- Inline script block in `<head>` is required to prevent screen flicker during hydration.

### Task 2: Hydration Script Injection in layout.js

**Files:**
- Modify: `src/app/layout.js`

**Interfaces:**
- Consumes: `localStorage.getItem('theme')` on client load.
- Produces: Synchronous inline script in HTML `<head>`.

- [ ] **Step 1: Inject hydration script inside Head element in layout.js**

Add the script block immediately at the top of the `<head>` children in `src/app/layout.js`:
```javascript
      <head>
        <script dangerouslySetInnerHTML={{ __html: `
          (function() {
            try {
              var theme = localStorage.getItem('theme');
              if (theme === 'dark') {
                document.documentElement.setAttribute('data-theme', 'dark');
              }
            } catch (e) {}
          })();
        ` }} />
        <link 
          rel="stylesheet" 
          href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css" 
          precedence="default"
        />
      </head>
```

### Task 3: State Management & Switch Button in Header.js

**Files:**
- Modify: `src/components/Header.js`

**Interfaces:**
- Produces: `isDarkMode` state, `toggleDarkMode` handler, and button UI elements.

- [ ] **Step 1: Define state and toggle functions inside Header component**

Add the state hook and handlers at the top of `Header` component inside `src/components/Header.js`:
```javascript
  const [isDarkMode, setIsDarkMode] = useState(false);

  useEffect(() => {
    try {
      const savedTheme = localStorage.getItem('theme');
      if (savedTheme === 'dark') {
        setIsDarkMode(true);
        document.documentElement.setAttribute('data-theme', 'dark');
      } else {
        setIsDarkMode(false);
        document.documentElement.removeAttribute('data-theme');
      }
    } catch (e) {}
  }, []);

  const toggleDarkMode = () => {
    const nextDark = !isDarkMode;
    setIsDarkMode(nextDark);
    try {
      if (nextDark) {
        localStorage.setItem('theme', 'dark');
        document.documentElement.setAttribute('data-theme', 'dark');
      } else {
        localStorage.setItem('theme', 'light');
        document.documentElement.removeAttribute('data-theme');
      }
    } catch (e) {}
  };
```
